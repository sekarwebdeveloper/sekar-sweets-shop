## 1. Database (Lovable Cloud / Supabase)

The backend is **Lovable Cloud** (Supabase under the hood). It is already provisioned and connected â€” no `.env` setup required in Lovable.

### Tables

#### `orders`
Stores the order header (one row per order).

| Column | Type | Notes |
|---|---|---|
| `id` | uuid | primary key |
| `order_number` | text | auto-generated `SS-YYMMDD-XXXXX` |

| `email`, `phone` | text | contact |
| `address`, `landmark`, `city`, `state`, `pincode` | text | delivery |
| `subtotal`, `delivery_charge`, `total` | numeric | amounts in INR |
| `payment_method` | text | default `COD` |
| `status` | text | default `pending` (you can update to `confirmed`, `shipped`, `delivered`, `cancelled`) |
| `notes` | text | optional admin notes |
| `created_at`, `updated_at` | timestamptz | |

#### `order_items`
One row per cart line item.

| Column | Type |
|---|---|
| `id` | uuid |
| `order_id` | uuid (â†’ orders.id, cascade delete) |
| `product_id`, `product_name`, `product_category`, `product_weight` | text |
| `unit_price`, `line_total` | numeric |
| `quantity` | integer |

### Row-Level Security
- **INSERT** is open to anonymous users (guest checkout).
- **SELECT / UPDATE / DELETE** are blocked from the client. View orders via:
  - Lovable â†’ **Cloud â†’ Database â†’ orders** table viewer
  - Or write an authenticated admin page later

### Viewing & exporting orders
1. Open Lovable â†’ Cloud (top tab) â†’ Database â†’ Tables â†’ `orders`
2. Click the **Export** button to download CSV
3. Or write a SQL query in the SQL editor

## 2. Vercel Deployment

### Connect repo
1. Push the project to GitHub (use the GitHub button in Lovable)
2. Go to https://vercel.com â†’ New Project â†’ Import the repo

### Build settings
Vercel will auto-detect Vite. Settings:
- **Build Command**: `npm run build`
- **Output Directory**: `dist`
- **Install Command**: `npm install`

### Environment variables (Vercel)
Copy these from your local `.env` (auto-generated by Lovable Cloud) into Vercel â†’ Project Settings â†’ Environment Variables:

```
VITE_SUPABASE_URL=...
VITE_SUPABASE_PUBLISHABLE_KEY=...
VITE_SUPABASE_PROJECT_ID=...
```

> âš ď¸Ź Only `VITE_*` variables are exposed to the browser. The `PUBLISHABLE_KEY` is safe to expose (it's the anon/public key).

### SPA routing
`vercel.json` is already configured to rewrite all paths to `index.html` so deep links (e.g. `/product/kaju-katli`) work on refresh.

### Custom domain
After deploy, go to Vercel â†’ Settings â†’ Domains â†’ add your domain (e.g. `sekarsweets.com`) and follow the DNS instructions.

## 3. Product Images

### Current state
Each product uses **5 placeholder AI-generated images** (the same set, reused) located in:
```
src/assets/products/sweet-placeholder-1.jpg
sweet-placeholder-2.jpg
sweet-placeholder-3.jpg
sweet-placeholder-4.jpg
sweet-placeholder-5.jpg
```

### Replacing with real client photos
Open `src/data/products.ts` and edit the `defaultGallery()` function, OR override per product:

```ts
import myKajuMain from "@/assets/products/kaju-katli-1.jpg";
import myKajuClose from "@/assets/products/kaju-katli-2.jpg";

// In the products array, override images for a specific product:
{
  ...,
  images: [
    { src: myKajuMain, alt: "Kaju Katli diamond pieces with silver vark" },
    { src: myKajuClose, alt: "Kaju Katli close-up showing texture" },
    // ... 3 more
  ],
}
```

**Image guidelines**:
- Square aspect ratio (1:1)
- 1024x1024 px or higher
- Save as `.jpg` (smaller files) or `.webp` (best compression)
- Always provide meaningful `alt` text (used for SEO + screen readers)

## 4. SEO Configuration

### Per-product SEO
Each product in `src/data/products.ts` has:
```ts
seoTitle: "Motichoor Laddu - Buy Online | Sekar Sweets",
seoDescription: "Golden pearl-shaped laddu...",
seoKeywords: ["motichoor laddu", "buy laddu online", ...],
```
Edit these directly per product.

### Per-page SEO
Each page (`Index`, `Shop`, `About`, `Contact`, `ProductDetail`) uses the `<SEO>` component. Edit the props inline:
```tsx
<SEO
  title="Your Title (under 60 chars)"
  description="Description (under 160 chars)"
  keywords={["keyword1", "keyword2"]}
  url="/your-path"
/>
```

### JSON-LD Schemas
Defined in `src/lib/seo-schemas.ts`:
- `organizationSchema` â€” Sekar Sweets brand info
- `websiteSchema` â€” site search action
- `productSchema(product)` â€” Product schema for each product page
- `breadcrumbSchema(items)` â€” breadcrumb trail

### Sitemap
Auto-generated at `public/sitemap.xml`. Re-generate after adding/removing products:
```bash
node scripts/generate-sitemap.mjs
```
Then redeploy.

### robots.txt
Located at `public/robots.txt`. Allows all crawlers, blocks `/checkout/*` and `/cart`.

### Open Graph image
Place a 1200x630 social share image at `public/og-image.jpg` for nice link previews on WhatsApp/Twitter/LinkedIn.

## 5. Common edits

### Add a new product
Edit `src/data/products.ts` â†’ add an entry to the appropriate category array â†’ run sitemap script â†’ redeploy.

### Change order status
Lovable Cloud â†’ Database â†’ `orders` â†’ click row â†’ edit `status` field. Or write an admin UI later.

### Change delivery threshold
Edit the line `const deliveryCharge = totalPrice > 500 ? 0 : 50;` in:
- `src/pages/CustomerDetails.tsx`
- `src/pages/Checkout.tsx`

### Add payment gateway later
The `handlePlaceOrder` function in `src/pages/Checkout.tsx` is the integration point. Insert your payment call before the Supabase insert, then mark `payment_method` and `status` accordingly.
